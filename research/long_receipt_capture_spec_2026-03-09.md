# 長いレシート撮影・認識仕様案

作成日: 2026-03-09

## 1. 推奨する撮影方法

### 基本方針
- 長いレシートは `上から下まで連続でなぞらない`
- `画面に収まる範囲` ごとに静止して撮る
- `上・中・下` を少し重ねながら撮る

### 推奨手順
1. レシート上部を画面いっぱいに入れる
2. 1秒前後止める
3. 少し下へずらして中段を撮る
4. 1秒前後止める
5. 同じように下段まで繰り返す

### 撮影ルール
- 真上から撮る
- 1カットごとに静止する
- 隣り合うカットは少し重ねる
- 文字が切れないようにする
- 影を入れない

## 2. 現状実装の制約

現在の実装は [video_processor.py](/D:/AI_PROJECT/ryosyusyo_ai/app/services/video_processor.py) で、1シーンにつき代表フレームを1枚だけ選んでいます。

そのため現状は:
- 長いレシートを分割して撮っても
- 最終的にはその中の1枚しか使われない

つまり、撮影方法としては正しいが、認識ロジック側がまだ追いついていません。

## 3. 実装方針

### 目標
- 長いレシートを `複数画像の1グループ` として扱う
- 上・中・下の情報を統合してOCR/AI解析する

### 方針
1. 同一レシートの連続フレームをグループ化
2. グループ内から複数の鮮明フレームを採用
3. OCRを各画像に実行
4. テキストを時系列に結合
5. 重複行を除去
6. 統合テキストをAIに渡す

## 4. 画面側仕様

### 顧客向け文言
- `長いレシートは、画面に収まる範囲ごとに止めて撮影してください`
- `上部・中段・下段を少し重ねながら撮ると認識しやすくなります`
- `上から下まで連続でなぞる撮影は避けてください`

### 結果画面
- 長いレシートは `1件` として表示
- ただし内部では複数画像を持つ
- 必要なら `上 / 中 / 下` を切り替えて確認できる

## 5. データ仕様

### 追加したい項目
- `receipts.group_id`
- `receipts.segment_index`
- `receipts.segment_count`
- `receipts.is_long_receipt`

または別テーブル:
- `receipt_segments`
  - `id`
  - `receipt_id`
  - `image_path`
  - `segment_index`
  - `ocr_text`
  - `frame_number`

## 6. 認識ロジック仕様

### シーン内での採用ルール
- 1グループ内で最大 `3〜5枚`
- 各採用画像は時間差を持たせる
- 鮮明度が低いものは除外
- 同じ内容すぎる画像は除外

### OCR統合
- 上から順に結合
- 類似行を除去
- 日付、合計、店舗名などのキー項目は最も信頼度が高いものを採用

### AI入力
- 画像1枚ではなく
  - `複数画像`
  - `結合済みOCR`
をセットで渡す
